# 문서 유형별 상세 템플릿

## 1. PRD 템플릿

### 제목 페이지
- 문서 제목 (H1, 24pt, Bold, 가운데 정렬)
- 작성자 / 날짜 / 버전 (본문, 가운데 정렬)
- 상태: Draft / Review / Approved (색상 강조)

### 본문 구조
```
[목차] — 헤딩에서 자동 생성

[1. Executive Summary] — H2
  요약 단락 2-3개

[2. 문제 정의 및 컨텍스트] — H2
  문제 설명 (Bold 처리)
  근거 및 데이터
  타겟 사용자 목록

[3. 목표 및 성공 지표] — H2
  목표 | 지표 | 목표값 | 일정 표

[4. 유저스토리 및 유스케이스] — H2
  번호 매긴 유저스토리 목록
  중첩 수락 기준

[5. 기능 요구사항] — H2
  ID | 요구사항 | 우선순위 | 상태 표

[6. 비기능 요구사항] — H2
  설명 포함 목록

[7. 범위 및 제약사항] — H2
  In Scope / Out of Scope 두 컬럼

[8. 디자인 고려사항] — H2
  디자인 결정 사항

[9. 런치 계획] — H2
  일정 표

[10. 위험 요소 및 완화] — H2
  위험 | 영향 | 확률 | 완화 방안 표

[11. 부록] — H2
  리서치 데이터 / 참고 문서
```

**포맷팅 규칙:**
- H1 (제목): 24pt, Bold, 진한 파란색 (#1a237e)
- H2 (섹션): 18pt, Bold, 진한 회색 (#424242)
- H3 (서브섹션): 14pt, Bold, 회색 (#616161)
- 본문: 11pt, Regular, 검정
- 표 헤더: 연회색 배경 (#f5f5f5), Bold
- 여백: 사방 1인치

## 2. 유저스토리 템플릿

### 제목 페이지
- 기능명 (H1)
- Epic/테마
- 스프린트 배정
- 작성자, 날짜

### 스토리 요약 섹션 (H2)
- 개요 단락
- 스토리 수 및 우선순위 분류

### 각 스토리 형식
```
[스토리 ID: 제목] — H3, 14pt, Bold, 파란색 (#1976d2)

As a [사용자] — 이탤릭체, 들여쓰기
I want [목표] — 이탤릭체, 들여쓰기
So that [혜택] — 이탤릭체, 들여쓰기

컨텍스트: — H4
[배경 단락]

수락 기준: — H4, Bold
• Given [전제 조건]
  When [액션]
  Then [기대 결과]

메모:
- 디자인: [링크] — 회색 (#757575)
- 의존성: [목록]
- 추정 작업량: [포인트]

─────────────────── (구분선)
```

## 3. 아이디어 검증 템플릿

### 제목 페이지
- 아이디어 제목 (H1)
- 제출 날짜
- 검증 상태
- 승인 결정 (크고 색상 강조)

### 아이디어 개요 (H2)
- One-liner 설명
- 해결하는 문제
- 타겟 사용자

### 검증 프레임워크 (H2)
**JTBD 분석 (H3):** Job Statement, 사용자 컨텍스트, 원하는 결과

**경쟁사 분석 (H3):** 경쟁사 | 솔루션 | 강점 | 약점 표

**가치 제안 (H3):** 차별화 포인트, 혜택

**DVF 검증 (H3):** 각 차원 점수 및 근거

**RICE 우선순위화 (H3):** RICE Score 강조 표시

**10문항 평가 (H3):** 질문 | 답변 | 점수 | 메모 표

### 권고사항 (H2)
- Go/No-Go 결정 — 대형 폰트, 색상 (Go: 초록, No-Go: 빨강)
- 근거 단락
- 다음 단계 목록
