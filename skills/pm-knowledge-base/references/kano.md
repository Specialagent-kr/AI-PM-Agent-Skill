# Kano 모델

## 개요

Kano 모델은 고객 만족도 기반으로 기능을 분류하는 프레임워크다.

---

## 3가지 카테고리

### 1. 기본 니즈 (Basic Needs / Must-haves)
- **정의:** 고객이 당연히 기대하는 기능. 있어도 만족 없고, 없으면 큰 불만족.
- **예시:** E-commerce 보안 결제 / 앱 안정성 / SaaS 데이터 보안
- **전략:** 반드시 포함, 과투자 불필요

### 2. 성과 니즈 (Performance Needs / Satisfiers)
- **정의:** 많을수록 만족도가 선형으로 증가하는 기능.
- **예시:** 빠른 페이지 로드 / 배터리 효율 / 더 많은 통합
- **전략:** 경쟁 차별화 요소, 비례 투자

### 3. 감동 니즈 (Excitement Needs / Delighters)
- **정의:** 예상치 못한 기능. 없어도 불만 없지만, 있으면 큰 만족.
- **예시:** 개인화 추천 / AI 기능 / 스마트 자동화
- **전략:** 혁신 기회, 성공 시 높은 ROI

---

## Kano 조사 방법

각 기능에 대해 2가지 질문을 한다:

**기능적 질문:** "이 기능이 있다면 어떻게 느끼십니까?"
**역기능적 질문:** "이 기능이 없다면 어떻게 느끼십니까?"

답변 선택지:
- 매우 만족한다 / 당연히 있어야 한다 / 상관없다 / 어쩔 수 없다 / 매우 불만이다

---

## 우선순위 결정 프로세스

1. **먼저:** 모든 기본 니즈(Must-have) 충족
2. **다음:** 성과 니즈(Performance)를 경쟁 수준까지 개선
3. **그 다음:** 감동 니즈(Delighter)로 차별화 추가

---

## Kano 템플릿

| 기능 | 카테고리 | 우선순위 | 근거 |
|------|----------|----------|------|
| 보안 결제 | 기본 니즈 | Must-have | 없으면 큰 불만족 |
| 빠른 결제 | 성과 니즈 | 높음 | 경쟁 차별화 요소 |
| 원클릭 구매 | 감동 니즈 | 보통 | 예상 외 높은 만족 |
| 다중 통화 | 성과 니즈 | 보통 | 옵션 증가 = 만족 증가 |

---

## 활용 시점

✅ 사용하기 좋은 상황:
- 기능 분류 및 우선순위 결정
- 고객 기대치 이해
- Must-have vs. 혁신 요소 균형
- 로드맵 계획
