# OKRs (Objectives and Key Results)

## 개요

OKR은 두 가지 요소로 구성된 목표 설정 프레임워크다:
- **Objective (목표):** 질적이고 영감을 주는 시간 제한 목표
- **Key Results (핵심 결과):** 성공을 측정하는 정량적 지표 (목표당 2-5개)

---

## OKR 구조

**Objective 작성 기준:**
- 질적이고 영감을 주는 표현
- 시간 제한 (분기 또는 연간)
- 야심 차지만 달성 가능
- 목표를 명확히 표현

**Key Results 작성 기준:**
- 구체적인 수치와 함께 측정 가능
- 목표당 2-5개
- 결과물(Output) 아닌 성과(Outcome) 중심
- 70% 달성도 성공으로 간주

---

## OKR 템플릿

```
Objective: [영감을 주는 목표 문장]

Key Result 1: [지표]를 [기준선]에서 [목표값]으로 [날짜]까지 달성
Key Result 2: [지표]를 [기준선]에서 [목표값]으로 [날짜]까지 달성
Key Result 3: [지표]를 [기준선]에서 [목표값]으로 [날짜]까지 달성
```

---

## OKR 예시

**제품 팀 OKR:**
```
Objective: 소규모 사업자를 위한 1위 모바일 결제 앱이 된다

Key Result 1: 월간 활성 사용자를 5만에서 10만으로 Q2까지 증가
Key Result 2: 앱스토어 평점을 4.0에서 4.5 이상으로 달성
Key Result 3: 결제 실패율을 3%에서 1% 미만으로 감소
Key Result 4: 신규 가맹점 파트너를 200개에서 500개로 확대
```

**기능 팀 OKR:**
```
Objective: 모바일 사용자를 위한 매끄러운 결제 경험을 제공한다

Key Result 1: 평균 결제 시간을 90초에서 30초로 단축
Key Result 2: 결제 완료율을 60%에서 85%로 향상
Key Result 3: 장바구니 이탈률을 15%에서 2%로 감소
```

## 좋은 OKR vs 나쁜 OKR

| ❌ 나쁜 OKR | ✅ 좋은 OKR | 이유 |
|------------|------------|------|
| O: 제품 개선 | O: 카테고리 최고 속도의 제품이 된다 | 구체적이고 영감 있음 |
| KR: 새 기능 출시 | KR: 기능 채택률을 40%로 높임 | 산출물 → 성과 |
| KR: 회의 10회 진행 | KR: 고객 만족도 8.5/10 달성 | 측정 가능한 성과 |
| KR: 성능 작업 | KR: 페이지 로드를 5초에서 2초로 단축 | 구체적 지표 |

---

## 활용 시점

✅ 적합:
- 분기/연간 목표 설정
- 팀을 회사 전략에 정렬
- 야심찬 목표 추적
- 우선순위 소통

❌ 부적합:
- 일상 업무 관리 (스프린트 사용)
- 개인 성과 평가 (별도 용도)
- 단기 목표 (1개월 미만)
