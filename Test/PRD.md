## 1. Executive Summary

### 제품 한 줄 요약
강의 상세페이지 상단에 Gen AI가 자동 생성하는 커리큘럼 요약 섹션을 추가해, 예비 수강생이 3초 내에 강의 핵심을 파악하고 탐색 의향을 높인다.

### 핵심 지표
- 강의 커리큘럼 탐색 비율: **3% → 7%** (+4%p)
- 강의 상세페이지 평균 탐색 시간: **10% 증가**

### 배경
고객 리서치 인사이트 *"커리큘럼을 먼저 이해하고 싶은데 찾기 불편하고 내용도 복잡해요"* 를 해결하는 Gen AI 자동화 기반의 UX 개선 프로젝트다.

## 2. 문제 정의 및 컨텍스트

### 문제 설명
패스트캠퍼스 강의 상세페이지에서 커리큘럼 정보는 페이지 하단 아코디언 구조로 배치되어 있다. 예비 수강생은 강의 내용을 파악하기 위해 긴 스크롤을 내려야 하며, 복잡한 섹션 구조로 인해 전체 강의 흐름을 한눈에 파악하기 어렵다.

### 근거 데이터

| 항목 | 내용 |
|------|------|
| **고객 리서치 VOC** | "커리큘럼을 먼저 이해하고 싶은데 찾기 불편하고 내용도 복잡해요" |
| **현재 탐색 지표** | 강의 커리큘럼 탐색 비율 3% (낮은 도달률) |
| **행동 패턴** | 상세페이지 방문 후 커리큘럼 섹션 도달 전 이탈 추정 |

### 타겟 사용자

| 구분 | 내용 |
|------|------|
| **주요 타겟** | 직장인 25~45세, 커리어 전환 또는 업스킬링 목적으로 강의 구매 검토 중인 예비 수강생 |
| **특성** | 시간이 부족해 빠른 정보 탐색 선호, 다수 강의를 비교·검토 |

### 현재 대안
강의 소개서 PDF 다운로드, YouTube 강의 미리보기, 수강후기 확인

**현재 대안의 한계**
- 강의 소개서 PDF: 다운로드 필요, 실시간 업데이트 반영 안 됨
- 커리큘럼 아코디언: 구조 파악에 시간 소요, 모바일 UX 불편

## 3. 목표 및 성공 지표

### 비즈니스 목표
강의 상세페이지에서 콘텐츠 탐색 행동을 늘려 구매 전환율 개선에 기여한다.

### 사용자 목표
예비 수강생이 강의 핵심 내용을 빠르게 파악하고, 내 목적에 맞는 강의인지 자신감 있게 판단할 수 있게 한다.

### KPI 목표

| KPI | 현재 | 목표 | 기간 | 측정 |
|-----|------|------|------|------|
| ■ **커리큘럼 탐색 비율** | 3% | 7% | 출시 후 4주 | GA 이벤트 추적 |
| ■ **상세페이지 평균 탐색 시간** | 기준값 | +10% | 출시 후 4주 | GA 세션 시간 |
| ■ **AI 요약 섹션 클릭률** (선택) | 0% | 15%+ | 출시 후 4주 | A/B 테스트 |

### 가드레일 지표 (하락하면 안 되는 지표)
- 페이지 로딩 속도 (LCP): 현재 대비 +200ms 이하
- 강의 구매 전환율: 유지 또는 향상

## 4. 유저스토리 및 유스케이스

### 주요 유저스토리

| ID | 스토리 |
|----|--------|
| **US-01** | 예비 수강생으로서, 강의 페이지 상단에서 주요 커리큘럼을 요약해 볼 수 있어서, 스크롤 없이 강의 내용을 빠르게 파악할 수 있다 |
| **US-02** | 예비 수강생으로서, 이 강의의 핵심 특징 3가지를 요약 형태로 확인할 수 있어서, 다른 강의와 빠르게 비교할 수 있다 |
| **US-03** | 강의 운영자로서, 커리큘럼이 업데이트될 때 요약 내용이 자동으로 갱신되어서, 별도의 수동 작업 없이 최신 정보를 유지할 수 있다 |

### 핵심 유스케이스 플로우

```
예비 수강생 → 강의 상세페이지 진입
    → [상단 개요 영역] AI 요약 섹션 노출
        → 강의 특징 3줄 요약 확인
        → 커리큘럼 핵심 섹션 5개 확인
    → 더 알고 싶으면 "전체 커리큘럼 보기" 클릭
    → 기존 커리큘럼 아코디언으로 스크롤
```

### 엣지 케이스
- 신규 강의 (커리큘럼 미등록): "커리큘럼 준비 중" 플레이스홀더 표시
- AI 생성 실패: 폴백으로 기존 커리큘럼 아코디언만 표시
- 강의 자료 업데이트: 24시간 내 자동 재생성

## 7. 범위 및 제약사항

### In Scope (이번 릴리즈 포함)
- 강의 상세페이지 상단 개요 영역 내 요약 섹션 추가
- Gen AI 자동 생성 파이프라인 (강의 커리큘럼 → 요약)
- A/B 테스트 설계 및 분석 도구 연동
- 파일럿: 선택된 강의 20개 적용 후 전사 롤아웃

### Out of Scope (이번 릴리즈 제외)
- 수강생 리뷰 기반 요약 (F-08)
- 개인화 추천 문구 (F-09)
- 모바일 앱 (웹 기반으로만 시작)
- B2B 기업 강의 페이지 (B2C 먼저 적용)

### 제약 조건
- **기술:** 기존 강의 상세페이지 CMS 구조 내에서 구현
- **운영:** AI 요약 초기 품질 검수를 위한 최소 인력 필요
- **일정:** 파일럿 포함 런치까지 약 8주 목표

## 8. 디자인 및 기술 고려사항

### UI/UX 가이드라인
- 상단 개요 영역 내 별도 카드 컴포넌트로 배치 (기존 영역 침범 최소화)
- 강의 특징: 아이콘 + 1~2문장 포맷 (3개)
- 커리큘럼 요약: 섹션명 + 핵심 키워드 리스트 (5개 이하)
- "AI가 요약한 내용입니다" 레이블 표시 (투명성)
- "전체 커리큘럼 보기" 링크로 기존 섹션 연결

### 아키텍처 개요

```
강의 데이터 변경 이벤트
        ↓
AI 요약 파이프라인 (배치 처리)
[커리큘럼 + 소개서 + 강의 자료]
        ↓
Gen AI API (Claude/GPT-4)
        ↓
요약 결과 캐시 저장 (DB/CDN)
        ↓
강의 상세페이지 API → 프론트엔드 렌더링
```

## 9. 런치 및 롤아웃 계획

| 단계 | 기간 | 주요 작업 |
|------|------|----------|
| **Phase 1 — 설계** | Week 1-2 | UI/UX 디자인, AI 프롬프트 설계, 데이터 파이프라인 설계 |
| **Phase 2 — 개발** | Week 3-5 | Gen AI 파이프라인 구현, 프론트엔드 컴포넌트 개발, A/B 테스트 설정 |
| **Phase 3 — 파일럿** | Week 6-7 | 선택 강의 20개 AI 요약 생성, 내부 품질 검수, A/B 테스트 시작 |
| **Phase 4 — 롤아웃** | Week 8+ | A/B 결과 기반 전사 적용 결정, 자동화 파이프라인 전환 |

### 파일럿 성공 기준
- 커리큘럼 탐색 비율 테스트 그룹 ≥ 5% (컨트롤 3%)
- AI 요약 품질 내부 평가 80점 이상 (5점 척도 평균)
- 페이지 로딩 속도 저하 없음

## 10. 위험 요소 및 완화 방안

| 위험 | 영향 | 확률 | 완화 방안 |
|------|------|------|----------|
| **위험 1** AI 요약 품질 낮음 (할루시네이션) | 높음 | 중간 | 실제 데이터 기반 프롬프트 설계, 초기 인간 검수 병행 |
| **위험 2** 강의 데이터 파이프라인 지연 | 중간 | 낮음 | 캐싱 + 배치 처리로 부하 분산 |
| **위험 3** A/B 테스트 표본 부족 | 중간 | 낮음 | 트래픽 높은 인기 강의로 파일럿 선정 |
| **위험 4** 사용자 AI 요약 신뢰도 낮음 | 중간 | 중간 | "AI 요약" 레이블 + 원본 커리큘럼 링크 제공 |
| **위험 5** 페이지 성능 저하 | 높음 | 낮음 | 정적 캐싱, CDN 배포, 비동기 로딩 |

### 미해결 질문
- Gen AI API: 내부 구축 vs. 외부 API (비용/성능 트레이드오프)
- AI 요약 재생성 트리거: 실시간 vs. 일 1회 배치
- 품질 기준: 자동 배포 전 사람 검수 기준 정의 필요
