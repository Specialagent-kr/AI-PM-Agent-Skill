> FastSummary 프로젝트 (패스트캠퍼스 강의 상세페이지 AI 커리큘럼 요약 섹션)

---

## 전체 플로우 개요

```
Human (PM)
    │
    │ 1. 아이디어 브리프 제출
    ▼
Claude Code (오케스트레이터)
    │
    ├─▶ [Skill] idea-generation (VALIDATION 모드)
    │       └─▶ [MCP] Google Drive — 검증 가이드 로드
    │
    ├─▶ [Skill] prd-documentation (WRITE 모드)
    │       └─▶ [MCP] Google Drive — PRD 템플릿 로드
    │
    ├─▶ [Skill] google-docs-writer
    │       └─▶ [MCP] Google Drive — 폴더 생성 + 문서 2개 생성
    │
    ├─▶ [Skill] presentation-generator
    │       └─▶ [MCP] Google Drive — Google Slides 18슬라이드 생성
    │
    ├─▶ [MCP] Google Drive — 두 문서 재작성 + 포맷 재적용
    │
    └─▶ [Local] 마크다운 파일 3개 생성
```

## 단계별 상세 시나리오

### STEP 1 — 아이디어 제출

| 항목 | 내용 |
|------|------|
| **행위자** | Human (PM) |
| **입력** | 아이디어 브리프 (역할, 목표, 인사이트, 아이디어, KPI, 참고 URL) |
| **출력** | Claude Code에 작업 지시 |

**요청 내용:**
- Role: 패스트캠퍼스 상품 탐색 PM
- Goal: 강의 상세페이지 콘텐츠 탐색 증가
- Insight: "커리큘럼 찾기 불편하고 내용도 복잡해요" (VOC)
- 아이디어: Gen AI 자동 생성 커리큘럼 요약 섹션을 상단에 추가
- KPI: 탐색 비율 3→7%, 탐색 시간 +10%

### STEP 2 — 아이디어 검증

| 항목 | 내용 |
|------|------|
| **행위자** | Claude Code + Skill (idea-generation) + MCP (Google Drive) |
| **스킬 모드** | VALIDATION |
| **입력** | PM 아이디어 브리프 |
| **출력** | 검증 보고서 (10문항 평가 + DVF + RICE + 권고) |

**세부 흐름:**

```
Claude Code
    │
    ├─ Skill: idea-generation 호출 (VALIDATION 모드)
    │
    └─ MCP: Google Drive
            └─ getGoogleDocContent
               (Product Idea Assessment Guide ID: 11sPz2NUU...)
               → 10가지 평가 질문 프레임워크 로드
```

**산출물:**
- Q1~Q10 평가: 5문항 ★★★★★, 4문항 ★★★★
- DVF: Desirability 5/5, Viability 4/5, Feasibility 4/5 → **총점 13/15**
- RICE Score: (30,000 × 2.0 × 0.80) / 2 = **24,000**
- 결정: **✅ GO**

### STEP 5 — PRD 발표자료 생성

| 항목 | 내용 |
|------|------|
| **행위자** | Human (PM) → Claude Code + Skill (presentation-generator) + MCP (Google Drive) |
| **트리거** | PM의 추가 요청: "PRD 발표를 위해 프레젠테이션 자료를 만들어줘" |
| **출력** | Google Slides 18슬라이드 |

**세부 흐름:**

```
Human → Claude Code
    │
    ├─ Skill: presentation-generator 호출 (PRD Review 타입)
    │
    └─ MCP: Google Drive
            ├─ createGoogleSlides (18슬라이드)
            │   → ID: 146jTYp95mYwXJrBY8d1b94JjVpdy_NqwL7ARtnb3Lbw
            │
            ├─ setGoogleSlidesBackground
            │   → 타이틀/클로징 슬라이드: 딥 블루 (#1a237e)
            │
            └─ formatGoogleSlidesText (×17 슬라이드 타이틀)
                → Bold, 32-44pt, 딥 블루 / 흰색
```

**슬라이드 구성 (PRD Review 18슬라이드):**
타이틀 → 목차 → 문제 정의 → 사용자 리서치 → 솔루션 개요 → 유저스토리 → 기능 요구사항 → 비기능 요구사항 → 성공 지표 → 디자인 목업 → 기술 아키텍처 → 구현 단계 → 리소스 요구사항 → 리스크/완화 → GTM 계획 → 타임라인 → 예산/ROI → 승인 요청

### STEP 6 — 포맷 오류 수정 및 재작성

| 항목 | 내용 |
|------|------|
| **행위자** | Human (PM) → Claude Code + MCP (Google Drive) |
| **트리거** | PM 피드백: "Google Docs 파일이 포맷이 깨져 — 텍스트 스타일 및 표 형식을 유지하도록 보완해줘" |
| **문제** | 파이프 테이블(`\| col \|`) → Google Docs에서 plain text로 렌더링됨 |
| **해결** | 파이프 테이블 제거 + 구조화된 레이블 텍스트로 재작성 + 헤딩/볼드 전체 재적용 |

**세부 흐름:**

```
Human → Claude Code
    │
    ├─ MCP: getGoogleDocContent (두 문서 현재 내용 확인)
    │
    ├─ MCP: updateGoogleDoc (아이디어 검증 보고서 전체 재작성)
    │   → 파이프 테이블 → 구조화 텍스트 변환
    │
    ├─ MCP: updateGoogleDoc (PRD 전체 재작성)
    │   → 파이프 테이블 → 구조화 텍스트 변환
    │
    ├─ MCP: formatGoogleDocParagraph (×21회 병렬 호출)
    │   → H1: 문서 제목 (양쪽)
    │   → H2: 아이디어 검증 8개 섹션 / PRD 11개 섹션
    │
    ├─ MCP: formatGoogleDocParagraph (×45회 병렬 호출)
    │   → H3: 아이디어 검증 15개 서브헤딩 (Q1~Q10, 역할/목표 등)
    │   → H3: PRD 29개 서브헤딩 (Must/Should/Nice, Phase 1~4 등)
    │
    └─ MCP: formatGoogleDocText (×46회 병렬 호출)
        → Bold: 경쟁사 이름, DVF/RICE 수치 (아이디어 검증)
        → Bold: F-01~F-09, US-01~03, 위험 1~5, KPI 항목 (PRD)
```

## 핵심 설계 원칙

### 1. Skill = 지식 레이어
스킬은 PM 방법론과 워크플로우 구조를 제공한다. 실제 실행(파일 생성, API 호출)은 Claude Code와 MCP가 담당한다.

### 2. MCP = 실행 레이어
Google Drive MCP는 실제 파일 시스템 작업을 수행한다. Claude Code는 MCP 도구를 직접 호출하며, 독립적인 호출은 병렬로 실행해 대기 시간을 최소화한다.

### 3. Claude Code = 오케스트레이터
Claude Code는 Human의 의도를 파악하고, 적절한 스킬을 선택해 호출하며, MCP 도구를 직접 조율한다. 이번 세션에서는 Sub Agent를 별도로 생성하지 않고 Claude Code가 모든 작업을 직접 처리했다.

### 4. 포맷팅 한계와 대응
Google Docs API는 마크다운 파이프 테이블(`| col |`)을 지원하지 않아 plain text로 렌더링된다. 이를 해결하기 위해 구조화된 레이블 텍스트 + 헤딩 계층(H1/H2/H3) + Bold 스타일을 조합하는 방식으로 시각적 구조를 표현했다.
