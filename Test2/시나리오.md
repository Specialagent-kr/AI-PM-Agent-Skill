# FastSummary v2 — Sub Agent 워크플로우 시나리오

> 3개의 Sub Agent를 활용한 PM 산출물 자동화 파이프라인

---

## 개요

FastSummary v2는 동일한 아이디어(FastSummary)를 **3개의 독립 Sub Agent**로 병렬/순차 처리하여 산출물을 생성한 실험적 접근이다. 이를 통해 **Sub Agent 활용 방식**과 **단일 Claude Code 컨텍스트 방식**(FastSummary v1)의 차이를 비교할 수 있다.

---

## 전체 워크플로우

```
Human (PM)
    │
    │ 아이디어 브리프 제공
    ▼
Claude Code (오케스트레이터)
    │
    ├──► Sub Agent 1 (idea-generation Skill)
    │         │ 아이디어 검증 수행
    │         │ Product Idea Assessment Guide 로드 (MCP Google Drive)
    │         │ 10문항 평가 + DVF + RICE 계산
    │         ▼
    │    ✅ 검증 결과 반환 (GO, DVF 12.5/15, RICE 140)
    │
    ├──► Sub Agent 2 (prd-documentation Skill)
    │         │ Sub Agent 1 결과를 입력으로 받아 PRD 작성
    │         │ PRD Template 로드 (MCP Google Drive)
    │         │ 11개 섹션 PRD 생성 (F-01~F-18, US-01~05)
    │         ▼
    │    ✅ PRD 반환
    │
    └──► Sub Agent 3 (Google Drive 업로드)
              │ Sub Agent 1+2 결과를 입력으로 받아 업로드
              │ FastSummary v2 폴더 생성 (MCP Google Drive)
              │ 아이디어 검증 보고서 Google Doc 생성 + 포맷팅
              │ PRD Google Doc 생성 + 포맷팅
              │ Google Slides 프레젠테이션 생성 (18슬라이드)
              ▼
         ✅ 3개 파일 Google Drive 업로드 완료
```

## 각 Sub Agent 상세

### Sub Agent 1 — 아이디어 검증

| 항목 | 내용 |
|------|------|
| **역할** | 아이디어 검증 (idea-generation Skill, VALIDATION 모드) |
| **입력** | Human의 아이디어 브리프 (역할, 목표, VOC, KPI) |
| **사용 도구** | MCP Google Drive (`getGoogleDocContent`) — Product Idea Assessment Guide 로드 |
| **처리 내용** | 10가지 기회 평가 질문 답변, DVF 점수 산출, RICE 점수 계산, Go/No-Go 결정 |
| **출력** | 아이디어 검증 보고서 (DVF 12.5/15, RICE 140, ✅ GO) |
| **실행 시간** | 약 2~3분 |

**Sub Agent 1 주요 결과:**
- DVF: Desirability 4.5 + Viability 4.0 + Feasibility 4.0 = **12.5 / 15**
- RICE: (350,000 × 2.0 × 0.80) / 4 = **140**
- 결정: **✅ GO** — 파일럿 진행 권장

---

### Sub Agent 2 — PRD 작성

| 항목 | 내용 |
|------|------|
| **역할** | PRD 작성 (prd-documentation Skill, WRITE 모드) |
| **입력** | Sub Agent 1의 검증 결과 + Human의 원본 브리프 |
| **사용 도구** | MCP Google Drive (`getGoogleDocContent`) — PRD Template 로드 |
| **처리 내용** | 11개 섹션 PRD 작성 (Executive Summary → 부록) |
| **출력** | 완성된 PRD (F-01~F-18, US-01~05, 위험요소 6개, 미해결 질문 3개) |
| **실행 시간** | 약 3~5분 |

**Sub Agent 2 주요 결과:**
- 유저스토리: US-01~US-05 (예비 수강생 3개 + 강사 1개 + 모바일 사용자 1개)
- 기능 요구사항: F-01~F-08 (P0) + F-09~F-13 (P1) + F-14~F-18 (P2)
- 롤아웃: Phase 1 (6주) → Phase 2 (8주) → Phase 3 (8주) = 총 22주

### Sub Agent 3 — Google Drive 업로드

| 항목 | 내용 |
|------|------|
| **역할** | 문서 생성 및 업로드 (google-docs-writer + presentation-generator Skill) |
| **입력** | Sub Agent 1+2의 전체 결과물 |
| **사용 도구** | MCP Google Drive (폴더 생성, Doc 생성, 포맷팅, Slides 생성) |

| **출력** | Google Drive에 3개 파일 (아이디어 검증 Doc, PRD Doc, 발표자료 Slides) |

## FastSummary v1 vs v2 비교

| 항목 | FastSummary v1 (단일 컨텍스트) | FastSummary v2 (3 Sub Agents) |
|------|-------------------------------|-------------------------------|
| **아키텍처** | Claude Code 단일 컨텍스트에서 순차 처리 | 3개 독립 Sub Agent + 오케스트레이터 |
| **Skill 활용** | ✅ Skill 지침 로드하여 직접 실행 | ✅ 각 Sub Agent가 전용 Skill 사용 |
| **컨텍스트 관리** | 모든 작업이 하나의 컨텍스트 | 각 Sub Agent가 독립 컨텍스트 보유 |
| **병렬 처리** | ✗ 순차 처리만 가능 | ✓ 독립 작업은 병렬 가능 (v2에서 순차) |
| **컨텍스트 오염** | 중간 산출물로 컨텍스트 누적 | 각 Agent가 목적에 집중된 클린 컨텍스트 |
| **실행 시간** | 약 30~40분 | 약 20~25분 (순차 실행 기준) |
| **실패 격리** | 한 단계 실패 시 전체 영향 | 각 Agent 독립 실패/재실행 가능 |
| **투명성** | 전체 과정이 하나의 스레드로 추적 | Agent별 독립 추적 (재개 ID 보유) |
| **총 MCP 호출** | 약 100+ 회 | Sub Agent 3만 138회 |
| **RICE Score** | 24,000 (산출 방식 A) | 140 (산출 방식 B) |
| **DVF 총점** | 13 / 15 | 12.5 / 15 |

> **RICE Score 차이 원인:** v1은 월간 Reach(30,000명) + Effort(2개월), v2는 분기 Reach(350,000명) + Effort(4개월)로 서로 다른 기준으로 산출. 절대값이 아닌 상대적 우선순위 비교에 활용해야 함.

## Sub Agent 패턴의 장점

### 1. 컨텍스트 집중
각 Sub Agent는 자신의 역할에만 집중하므로, 불필요한 중간 산출물로 컨텍스트가 오염되지 않는다. Sub Agent 2는 PRD 작성에만, Sub Agent 3는 업로드에만 집중한다.

### 2. 실패 격리 및 재실행
Sub Agent 3이 중간에 실패하더라도 Sub Agent 1과 2의 결과는 보존된다. 각 Agent는 고유 ID(agentId)로 재개(resume)할 수 있다.

### 3. 확장성
향후 Sub Agent를 추가하거나 병렬로 실행하기 쉽다. 예: Sub Agent 4(유저스토리 작성), Sub Agent 5(로드맵 생성)를 병렬로 추가 가능.

### 4. 역할 명확성
오케스트레이터(Claude Code)는 조율에만 집중하고, 실제 전문 작업은 각 Sub Agent에게 위임한다.
