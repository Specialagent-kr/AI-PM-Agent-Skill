## 1. Executive Summary

### 제품 개요
FastSummary v2는 강의 상세페이지 상단 개요 영역에 Gen AI가 자동 생성한 커리큘럼 요약 섹션을 삽입하는 기능이다. 예비 수강생이 30초 이내에 강의 핵심을 파악하고 탐색 의향을 높이는 것을 목표로 한다.

### 핵심 지표
- 커리큘럼 탐색 비율: **3% → 7%** (+4%p)
- 상세페이지 → 장바구니 전환율: **+5%p**
- AI 요약 생성 성공률: **95%+**

### 배경
고객 리서치 인사이트 *"커리큘럼을 먼저 이해하고 싶은데 찾기 불편하고 내용도 복잡해요"*를 해결하는 Gen AI 자동화 기반 UX 개선 프로젝트. DVF 12.5/15, RICE 140으로 즉시 추진 권고.

## 2. 문제 정의 및 컨텍스트

### 문제 설명
강의 상세페이지 커리큘럼 정보가 하단 트리 구조로 배치되어 있어, 예비 수강생이 강의 핵심 내용을 빠르게 파악하지 못한 채 이탈하는 문제 발생. 커리큘럼 섹션 스크롤 도달 비율: 3% (내부 데이터).

### 근거 데이터

| 항목 | 내용 |
|------|------|
| **고객 리서치 VOC** | "커리큘럼을 먼저 이해하고 싶은데 찾기 불편하고 내용도 복잡해요" |
| **현재 탐색 지표** | 커리큘럼 섹션 도달 비율 3% |
| **행동 패턴** | 상세페이지 방문 후 커리큘럼 도달 전 이탈 추정 |

### 타겟 사용자

| 구분 | 내용 |
|------|------|
| **주요 타겟** | 직장인 25~45세, 커리어 전환 또는 업스킬링 목적 예비 수강생 |
| **특성** | 시간 부족, 빠른 정보 탐색 선호, 다수 강의 비교·검토 |

### 현재 대안
- 현재 커리큘럼 섹션: 하단 위치, 트리 구조, 접근성 낮음
- 강의 소개 영상: 일부 강의만 존재
- Udemy/Coursera: 상단에 학습 내용 섹션 있으나 수동 작성

## 3. 목표 및 성공 지표

### 비즈니스 목표
- 강의 상세페이지 콘텐츠 탐색 행동을 늘려 구매 전환율 개선에 기여한다.
- Gen AI 자동화로 운영 인력 없이 전 강의 커리큘럼 요약을 제공한다.

### 사용자 목표
예비 수강생이 30초 이내에 강의 핵심 학습 내용을 파악할 수 있게 한다.

### KPI 목표

| KPI | 현재 | 목표 | 기간 | 측정 |
|-----|------|------|------|------|
| ■ **커리큘럼 탐색 비율** | 3% | 7% | 파일럿 6주 | GA4 스크롤 이벤트 |
| ■ **상세페이지 체류 시간** | 기준값 | +10% | 파일럿 6주 | GA4 세션 시간 |
| ■ **장바구니 전환율** | 현재값 | +5%p | Phase 2 | GA4 전환 퍼널 |
| ■ **AI 요약 생성 성공률** | N/A | 95%+ | 런칭 즉시 | 서버 로그 |

### 가드레일 지표 (하락하면 안 되는 지표)
- 부정확 신고 비율 5% 초과 시 즉시 중단
- 강사 이의 제기 10건/주 초과 시 프로세스 재검토
- 페이지 로딩 지연 3초 초과 시 캐싱 전략 즉시 적용

## 4. 유저스토리 및 유스케이스

### 주요 유저스토리

| ID | 스토리 |
|----|--------|
| **US-01** | 예비 수강생으로서, 상세페이지 상단에서 핵심 커리큘럼을 요약해 볼 수 있어서, 스크롤 없이 강의 내용을 빠르게 파악할 수 있다 |
| **US-02** | 예비 수강생으로서, 요약에서 더 알고 싶을 때 클릭 한 번으로 전체 커리큘럼 섹션으로 이동할 수 있어서, 추가 탐색이 편리하다 |
| **US-03** | 강의 운영자로서, 커리큘럼이 업데이트될 때 요약이 자동으로 갱신되어서, 수동 작업 없이 최신 정보를 유지할 수 있다 |
| **US-04** | 강사로서, AI 요약이 부정확할 때 수정을 요청할 수 있어서, 내 강의 정보가 정확하게 전달된다 |
| **US-05** | 모바일 사용자로서, 스마트폰에서도 AI 요약을 깔끔하게 볼 수 있어서, 이동 중에도 강의를 탐색할 수 있다 |

### 핵심 유스케이스 플로우

```
예비 수강생 → 강의 상세페이지 진입
    → 상단 Fold에서 AI 요약 섹션 노출
        → 핵심 포인트 3~5개 확인
    → 관심 있으면 "전체 커리큘럼 보기" 클릭
    → 구매 신청 또는 이탈
```

### 엣지 케이스
- 커리큘럼 데이터 없음 → 요약 섹션 미표시 (graceful degradation)
- AI 생성 실패 → 섹션 미표시, 오류 로그 기록
- 커리큘럼 업데이트 → 24시간 내 자동 재생성

## 7. 범위 및 제약사항

### In Scope (이번 릴리즈 포함)
- 파일럿 상위 100개 강의 AI 요약 섹션 표시
- 강의 커리큘럼 텍스트 기반 자동 요약 생성
- 상단 Fold 영역 UI 컴포넌트 개발
- GA4 이벤트 추적 및 KPI 대시보드
- 사용자 피드백(신고) UI
- A/B 테스트 설정 (Phase 2)

### Out of Scope (이번 릴리즈 제외)
- 강의 영상 콘텐츠 분석 (텍스트만 사용)
- 실시간 요약 재생성 (배치 기반)
- 강사 직접 편집 인터페이스 (Phase 2 이후)
- 개인화 요약 (Phase 3)
- 타 플랫폼(앱) 배포

### 제약 조건
- **기술:** 기존 상세페이지 프론트엔드 아키텍처 내 컴포넌트 추가 방식
- **운영:** 파일럿 기간 사후 고지, Phase 2부터 강사 사전 동의 프로세스
- **일정:** 파일럿 포함 6주 목표
- **예산:** Claude API 파일럿 비용 약 $1~3 추정

## 8. 디자인 및 기술 고려사항

### UI/UX 가이드라인
- 상단 Fold 영역, 타이틀과 수강 신청 버튼 사이 배치
- 3~5개 핵심 학습 포인트 불릿 형태
- "AI가 요약한 내용입니다" 뱃지 표시 (투명성)
- "전체 커리큘럼 보기" 앵커 링크
- 모바일: 2~3개 기본 표시 후 "더 보기" UX 고려

### 아키텍처 개요

```
강의 커리큘럼 DB
        ↓
Batch Job Scheduler (매일 새벽 2시)
        ↓
Summary Generator (Claude API 호출)
        ↓
요약 저장 (DB + Redis 캐싱, TTL 24h)
        ↓
상세페이지 API → 프론트엔드 렌더링
```

### 기술 스택

| 구성 요소 | 기술 |
|-----------|------|
| **AI API** | Claude API (claude-3-5-haiku, 비용 효율) |
| **캐싱** | Redis (24시간 TTL) |
| **저장소** | 기존 RDB (summary 테이블 추가) |
| **프론트엔드** | React 컴포넌트 추가 |
| **스케줄러** | Celery + Cron 또는 AWS Lambda |
